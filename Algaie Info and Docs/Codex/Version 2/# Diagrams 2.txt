## Diagram 3 — Control flow / orchestration
```mermaid
sequenceDiagram
  participant U as User/Runner
  participant E as Entrypoint
  participant C as Config Loader
  participant D as Data/Features
  participant S as Strategy/Decision
  participant R as Risk/Constraints
  participant B as Broker
  participant T as Telemetry
  U->>E: start run
  E->>C: load config
  E->>D: build universe/features
  D->>S: feature frame
  S->>R: generate decisions
  R->>B: constrained orders
  B->>T: fills/status
  E->>T: run summary
```
